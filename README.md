# 钢筋感知项目 — 论文方向与近期路线

写给一起做这个项目的伙伴们。这份文档把我们目前的处境、要走的方向、
还差的事情整理成一份备忘录。

---

## 1. Risky point

2025 年已经有一篇论文做了 rebar detection and instance segmentation
的公开 benchmark（数据集 ROI-1555 已经放在 Hugging Face），覆盖了不同
钢筋类型、相机视角、排布模式、装配阶段，并评估了 6 个目标检测方法和
4 个实例分割方法。

数据集地址：[ROI-1555 on Hugging Face](https://huggingface.co/datasets/SYliuuuuuu/ROI-1555_Rebar_Detection_and_Instance_Segmentation_Dataset)

不过它的关注点是各种钢筋形态和角度（比如矩形钢筋、半成品钢筋笼），
而 **我们关注的是钢筋网平面本身，以及在其之上的结构推理与量测**。
方向不同，但 detection 这条赛道已经被占住了，论文必须从结构推理这一
侧切入。

---

## 2. 核心论点

一句话：

> 我们在 **度量平面空间**（metric plane-space）里聚类钢筋绑扎点，
> 而不是在像素空间里 — 再用网格拓扑把缺失交点补回来、把误检剔出去，
> 从而把 SAM/YOLO 输出的零散候选转换成可量测、可验收的钢筋网。

---

## 3. 现有模块

外面看我们的 pipeline 容易看成 "SAM + YOLO + RANSAC + 拟合" 的工程
拼装，这是论文里要避开的写法。在论文里，要把同样的代码讲成三层：

| 层级 | 作用 | 论文里的角色 |
|------|------|--------------|
| 视觉感知层 | 哪些像素可能是钢筋 / 交点 / 边界 — SAM 掩码、YOLO 关键点、候选区域 | **候选生成器**，不是创新点 |
| 几何约束层 | 把候选投影到平面 / 平行 / 网格结构上，剔除几何上不一致的候选 | 核心创新 — RANSAC 平面、双方向直线拟合、度量重投影 |
| 结构推理层 | 恢复钢筋线实例、交点、网格排序、间距、被遮挡的交点 | 第二创新点 — 拓扑恢复 |

写作上的铁律：**不要把论文写成 "我们用了 SAM + YOLO + RANSAC"。**
要写成 "我们仅把通用视觉模型作为候选生成器，依靠显式的几何与拓扑
约束来恢复度量上有效的钢筋网。"

流程图大概长这样：

```
输入: RGB + 双目深度 / 点云
        │
        ▼
视觉候选生成
  (SAM 掩码 + YOLO 交点)
        │
        ▼
平面约束筛选
  (RANSAC + 投影)
        │
        ▼
主方向直线拟合
  (双方向，度量平面空间)
        │
        ▼
拓扑感知网格推理
  (交点恢复 + 离群剔除 + 排序)
        │
        ▼
度量验收输出
  (间距、偏差、缺筋、合格/不合格)
```

## 4. 代码 details

把整条 pipeline 拆开看，每个 stage 在 §3 的 "三层定位" 里都对应一个
明确角色。下面按执行顺序把每一步的 *作用*、*关键设计* 和 *它在整体
论点中起什么作用* 各写一段。

### 4.1 SAM 掩码处理（`stages/sam_mask.py`）

**作用。** 把 SAM 输出的掩码（可能是 RGBA、单通道、二值，分辨率也不
一定和原图对齐）规范化成与底图同尺寸的二值掩码，并据此生成一张
"前景保留、背景灰化" 的分割图作为后续 stage 的输入。

**关键设计。** 这一步本身没有创新，但它决定了下游能看到的像素范围 —
背景灰化（240,240,240）等于给 YOLO 关掉了大部分干扰目标。在论文里
这一层属于 "视觉感知层 / 候选生成"，是 commodity，不能当卖点。

**对整体论点的贡献。** 让 YOLO 只在 "可能是钢筋" 的像素上工作，等价于
把通用检测器局部化到我们关心的语义区域，把后续几何约束的输入信噪
比拉高。

---

### 4.2 平面提取（`stages/plane_extraction.py`）

**作用。** 在 SAM 掩码覆盖的点云子集上拟合一个主平面，再把所有距离该
平面 ±阈值（默认 ±3 cm）以内的像素保留下来，得到一张 *几何精化后*
的 refined mask。

**关键设计有三处。**

1. **双轨道拟合：SVD-from-seeds 优先，RANSAC 兜底。** 当用户在 SAM
   交互时点过几个种子点（≥3 个有效 3D 坐标），直接用这些点的 SVD
   解析平面；只在缺少种子时才退回 RANSAC。这对应"用户意图是最强的
   先验"这个直觉 — 工程上点过的位置就是真正想测的那一面。
2. **Proximity envelope。** 不是对称的 ±阈值，而是按种子点在平面法
   向上的实际分布算一个 [near, far] 区间（near 多留 0.5 cm，far 多留
   1.5 cm），允许"靠相机这一侧严一点、远一侧宽一点"的非对称容差。
   这一段是处理 *双层钢筋干扰* 的关键 — 第二层钢筋通常在远侧，可以
   用 envelope 自然剔除。
3. **RANSAC 后的 SVD 二次精化。** 找到 best-inlier set 之后再做一次
   SVD，避免最佳采样的三个点本身有噪。

**对整体论点的贡献。** 这是论文里 "plane-constrained filtering" 的
实现，也是 §3 表里 "几何约束层" 的第一根支柱。它解决的具体 failure
mode：背景误检、第二层钢筋干扰、SAM 漏框出来的边缘。Ablation 表里
`w/o plane constraint` 这一行就在测它。

---

### 4.3 YOLO 关键点检测（`stages/knot_detection.py`）

**作用。** 把分割后的图像发给远端 YOLO 服务，拿回一组以 4-keypoint
四边形姿态（pose data）描述的绑扎点候选。

**关键设计。** 这一 stage 在代码上很薄 — 它就是个 HTTP client，加
上一个 "如果旁边已经有标注就直接复用" 的小开关。重点是 **我们故意
把检测限制为 "候选生成"**，不让它直接产出最终结果。

**对整体论点的贡献。** YOLO 输出在论文里只能算 "noisy proposals"。
我们承认它会漏检（被遮挡的交点没看见）、会误检（绑扎丝、阴影、纹理
被识别成绑扎点）。论文的故事正是建立在 "原始 detection 不够好，所以
我们用几何约束去清洗它" — 这一层越是不动声色越好，因为它越像
commodity，几何层的贡献就越突出。

---

### 4.4 直线拟合（`stages/line_fitting.py`） — **最核心的一步**

这一步代码最长、也是论文最值得讲的 stage。它有两种工作模式，由可用
的输入决定：

#### (a) 度量平面空间模式（**核心创新**）

当 plane metadata、深度图、相机内参都齐时，进入这一支：

1. 把每个 YOLO 关键点像素 (u, v) 还原成 3D 点 P。优先用深度图（如
   像素有有效 z）；没有深度时退回 "射线和平面求交" — 即沿相机光线
   方向找它和已拟合平面的交点。
2. 把 P 投影到平面上：`P' = P − (n·P + d) n`。
3. 在平面上建一个正交基 (e1, e2) — `e1` 取世界 x 轴在平面上的投影，
   `e2 = n × e1`。所有 3D 点用这个基底压成 2D 坐标 (s, t)，**单位是米**。
4. 在这个度量空间里聚类、拟合直线。
5. 拟合完再把直线两端点反投影回图像坐标，用于可视化和后续比对。

**为什么这是核心创新。** 像素坐标受透视影响 — 同一根钢筋在画面前
后端的间距不一样、不同视角下的 mAP 也不可比。一旦切到度量平面坐标，
"间距" 就是真实物理距离，对相机视角、距离、轻微倾斜都不敏感。
这是我们能写 "spacing MAE in millimetres" 而不是 "pixel error" 的
唯一原因。

#### (b) 像素空间 + Hough 主方向先验

平面信息齐但内参/深度缺失时，仍可在像素空间聚类，但不是无方向的 —
而是先在 refined mask 上跑 Canny + HoughLinesP，对所有线段角度做
直方图聚类（`_cluster_dominant_angles`），挑出 1–2 个主方向。后续
聚类用这个角度作为 "钢筋应该沿这个方向延伸" 的先验。

**关键设计：垂直投影聚类（`_fit_lines_by_perpendicular_projection`）。**
对每个候选主方向，把所有关键点投影到它的法向轴上得到一组 1D 值，
然后做 *gap-adaptive* 一维聚类 — 用 sorted-gap 序列里 "膝点" 的
几何均值作为分簇阈值，能自适应不同密度的钢筋网。

近 0°/90° 的方向走另一条更稳的路线：构建有向最近邻图，用宽高比
约束（`aspect = 5.0`）确保只有 "沿主方向接近、法向位移很小" 的两个
点才合并，最后用 union-find 收簇。

#### 自适应距离阈值（在 (a) 和 (b) 都有）

`_estimate_metric_max_dist` 用所有点对的 *最近邻距离中位数* 估计典型
间距，把 cluster 半径设为 3.5 倍中位数。这让算法不需要预设 "钢筋
间距 = 多少 mm"，能直接吃不同规格的工地数据。这一点对论文 "跨场景
泛化" 的论述很关键。

#### 双向分类（`_classify_lines_hv`）

两组主方向拟合出的直线，按 "哪组离水平更近" 决定谁是 horizontal、
谁是 vertical，然后各自按中点坐标排序。这一步目前只是 *排序*，
**还没有做网格索引** — 这就是 §5 推动的 grid_topology 模块要补的
缺口。

**对整体论点的贡献。** 整个 §4.4 是 "几何约束层" 的第二根支柱，
也是论文 contribution 的实质所在。Ablation 里 `w/o metric projection`
和 `w/o dominant direction` 两行直接对应它的两个设计选择。

---

### 4.5 空间分析（`stages/spatial_analysis.py`）

**作用。** 在拟合到的直线上调用 `analyzer.calculate_3d_spatial_metrics`
计算 3D 空间间距和均匀度（mean / std → uniformity = 1 / (1 + σ/μ)）。

**关键设计 / 当前状态。** 真正算 3D 间距的 `calculate_3d_spatial_metrics`
依赖外部 AIMatcher（双目特征匹配）— 这个项目里目前是 stub。一旦
AIMatcher 接上，结合 §4.4 已经算好的度量平面坐标，可以直接得到
mm 级的 spacing。

**对整体论点的贡献。** 这一层是 "metric inspection output" 的入口 —
最终交付给土木侧的就是这里的均匀度、间距均值/方差、合格判定。论文
"可量测、可验收" 的最后一句话由它兑现。

---

### 4.6 可视化（`stages/visualization.py`）

**作用。** 在矫正后的底图（或分割图）上画出关键点和拟合直线，叠加
H/V 数量计数；可选地拼出 before/after 的对比图。

**对整体论点的贡献。** 这一层在论文里只用来出 figure，不算贡献。
但它是 demo / 现场验收 UI 的 backbone，工程价值不能丢。

## 5. 实验TODO

### 5.1 Baseline

三个对比方法：

1. **YOLO-only**：仅检测交点，朴素最近邻算间距。
2. **SAM-only + 骨架化 + 直线拟合**：纯分割路线。
3. **Hough / RANSAC line detection**：传统直线检测。


### 5.2 Ablation — 每行对应一个具体的 failure mode

| 变体 | 关掉什么 | 证明什么 |
|------|----------|----------|
| `w/o plane constraint` | 平面筛选 | 平面约束减少背景误检 |
| `w/o metric projection` | 度量平面空间聚类 | **核心 ablation** — 度量坐标在透视下优于像素坐标 |
| `w/o dominant direction` | Hough 主方向先验 | 方向先验有助于直线恢复 |
| `w/o topology correction` | 网格拓扑推理 | 拓扑能补回被遮挡的交点 |
| `w/o SAM` | 只用 YOLO | 分割有助于直线定位 |
| `w/o YOLO` | 只用 SAM | 关键点检测有助于交点定位 |

`w/o metric projection` 这一行最重要 — 它直接证明我们的核心论点。
